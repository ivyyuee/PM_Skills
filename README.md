# PM Skills

这个仓库用于存放可复用的产品管理类 Claude Skills。

## PRD-review

`PRD-review` 是一个中文优先的 **PRD 预评审 Skill**，适合在正式研发评审、设计评审或老板过会前，先把需求文档里的关键缺口找出来并补齐。

它不是简单地“一次性给一堆评语”，而是默认采用 **渐进式提问**：
- 先帮你确认对 PRD 的理解
- 再做一轮完整性扫描
- 然后一次只聚焦一个最大缺口
- 逐步把指标、边界、依赖、异常场景、埋点口径等内容问清楚
- 在信息足够时，直接生成可回填到 PRD 的文字

## 适用范围

这个 Skill 特别适合 **互联网医疗健康类产品** 的 PRD 评审，例如：
- 在线问诊、挂号、处方、随访、医患沟通
- 体检、检验、报告查看、报告解读、健康档案
- 健康管理、疾病管理、会员服务、分层运营
- 服务预约、采样寄送、履约链路、健康商城
- 含 AI 解读、智能问答、风险提示等健康服务功能

同时，它也可以作为一个更通用的 PRD 评审框架，用于其他产品场景，只是会少一些医疗健康领域特有的追问。

## 这个 Skill 能帮你做什么

- 判断一份 PRD 是否已经足够清晰、完整、可落地
- 发现文档中的高风险缺口，例如：
  - 成功指标缺失
  - 权限边界不清
  - 埋点口径没定义
  - 依赖和风险没写
  - 待确认问题没有集中管理
- 用技术、设计、测试、业务四个视角提前 challenge 文档
- 把讨论结果整理成可直接回填的 PRD 文案

## 如何使用

### 1. 安装 Skill

把仓库里的文件复制到你的 Claude Code 命令目录：

- 源文件：`skills/PRD-review.md`
- 目标路径：`~/.claude/commands/PRD-review.md`

### 2. 调用方式

在 Claude Code 中使用：

```text
/PRD-review
```

然后给它以下任一种输入：
- 一个 PRD 文件路径
- PRD 的关键章节
- 一段需求背景 + 原型说明
- 一份你想在评审前先查缺补漏的需求文档

### 3. 默认交互方式

Skill 会先让你选择进入方式：

1. **逐轮引导模式**：一轮一轮带你审，适合打磨文档
2. **直接完整评审**：一次给完整结论，适合赶时间
3. **基于现有材料先判断**：适合上下文不完整时先快速看问题

如果你不指定，它会默认推荐 **逐轮引导模式**。

## 使用示例

### 示例 1：先走渐进式评审

```text
/PRD-review /Users/you/Desktop/某功能PRD.md
```

### 示例 2：让它直接给完整评审

```text
/PRD-review
不要多轮，直接帮我评审这份 PRD，告诉我能不能拿去给研发评审。
```

### 示例 3：只补最关键缺口

```text
/PRD-review
帮我看这份 PRD 里最需要补的是指标、权限还是埋点，我们一个一个来。
```

## 适合什么样的输入材料

效果最好的输入通常包括：
- PRD 正文
- 原型图或交互说明
- 业务背景
- 当前已知目标指标
- 已知的边界、依赖或待确认事项

如果材料还不完整，也可以直接开始；Skill 会通过提问帮你把缺口逐步补上。

## 输出结果通常包括

- 紧凑版完整性扫描
- 最大缺口判断
- 渐进式补齐后的规则/指标口径
- 角色化评审意见
- 可直接回填 PRD 的修改建议
- 上会前必须补充的内容清单

## 适合谁使用

- 产品经理
- 需求分析师
- 设计负责人
- 技术负责人
- 测试负责人
- 在正式评审前想先提升 PRD 质量的跨职能团队

## 说明

这个仓库中的 `PRD-review` 是一个通用版、脱敏版评审 Skill。
如果你的团队有更强的行业、产品或公司背景知识，也可以在此基础上继续做定制化扩展。
